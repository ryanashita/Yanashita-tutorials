# Generating Three-Address Code, Performing Liveness Analysis, and Allocating registers
This is a tutorial on how to generate three-address code (TAC) from an AST, perform liveness analysis on the temporaries generated by the TAC, and then allocate registers for the temporaries.
The motivation behind this tutorial and its contents is part my independent study in building a compiler that compiles custom programs to PTX -Parallel Thread Execution (PTX) is Nvidia's low-level virtual machine and ISA acting as a stable IR for CUDA-.
This tutorial is a continuation of this series, and is the logical next step after the previous tutorial about writing a parser and AST for the compiler's domain specific language (DSL). The ultimate goal, as I've said in previous tutorials, is to execute custom fatbin files generated by my compiler using the Nvidia Driver API.
I hope that this tutorial is helpful.

All files this tutorial refers to will be in the ~/toy-gpu-compiler-frontend/ directory.
