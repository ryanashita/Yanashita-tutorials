# Writing a Parser and AST for custom GPU kernel Domain Specific Language
This is a tutorial on how to write a custom parser using the taocpp/PEGTL library and how to build custom nodes that PEGTL can use to generate an AST. The motivation behind this tutorial and its contents is part of my independent study in building a compiler that compiles custom programs to PTX -*Parallel Thread Execution (PTX) is Nvidia's low-level virtual machine and ISA acting as a stable IR for CUDA*-. 

In my previous tutorial (~/grafting-custom-gpu-binaries/00-tutorial.md) I grafted custom GPU sections into a fatbin file and executed fatbins with the Nvidia Driver API. This tutorial builds the compiler with the contents of the first tutorial in mind, with the ultimate goal of executing custom fatbin files generated by the compiler. 

An additional motivation for this tutorial is to help myself remember the design choices of the AST, and the specific C++ syntax that makes the AST possible (hint: it has everything to do with OOP). 

All files I will be referring to and pasting code snippets of will be in the ```~/toy-gpu-compiler-frontend/``` directory. 

## Understanding taocpp/PEGTL and Building a Parser
I decided to build my parser using a parsing expression grammar (PEG) library. A parse expression grammar is a way to define a parser, and it "describes a formal language in terms of a set of rules for recognizing strings in the language"([PEG Wikipedia]([https://www.google.com](https://en.wikipedia.org/wiki/Parsing_expression_grammar))). and it is deterministic and unambigous by design.
